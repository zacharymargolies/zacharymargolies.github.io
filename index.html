fig

Your classroom should be a reflection of your personal learning style. So we made classrooms customizable.

Fig rooms provide students with a modularized real-time collaborative learning environment. Fig allows students to build a virtual ‘classroom’ where they can use built-in learning modules to work and study together on a variety of subjects.

Since real-time collaboration is integral to our platform, we leveraged Google Firestore’s listeners and callbacks to instantly sync application data across users. Data was modeled shallowly for read efficiency and scalability. Our stack included Express and the Twilio API for video chat capabilities.

On the front-end, we used React for its modularity and reusability. In developing the canvas module, we ran into the challenge of optimizing rendering and decided to batch the HTML Canvas micro-strokes (pixel to pixel) as larger macro-strokes objects to elevate the strain on database reads, which improved performance.

Another consideration during development was protecting code editors against malicious or non-terminating code. We set up execution time-outs and sanitized code before evaluation. Material UI and React Drag and Drop libraries were used to help provide a clean aesthetic and great fig user experience.

  • Technology Stack:
  • Google Firebase Firestore
  • React
  • Express
  • Semantic UI / Material UI

EYEZ

A staring contest where you can play anyone in the world.

EYEZ was built using React Native in conjuction with the Expo XDE. EYEZ is a project built over the course of a 4 day Stackathon. It's purpose is to break the physical barrier of a staring contest and allow users to play anyone in the world.

I achieved this through use of the True Depth sensor on the iPhone X series. Expo helped provided access and helped me leverage these native device features with ease. The True Depth sensor provides 50 blendshape coeffecients which map the expression of the human face. I used these coeffecients to tell whether a person was blinking or smiling.

Socket.io was used to enable users to compete realtime. When a combination of blendshapes indicates that a user is smiling or blinking (fulfilling the lost condition) an event is emitted to the players in the room to update them of the change in game status. Users also have the ability to send one-time powerups in the form of comedic short clips to try and distract their opponent and make them laugh to tip the game in their favor.

  • Technologies Used:
  • React Native
  • Socket.io
  • iPhone X True Depth Sensor

Vroom

Your one-stop shop for all things luxury cars.

Vroom is an ecommerce site built on the NERD stack. Vroom offers customers a beautifully designed shopping experience, as well as fully fledged backend administrator capabilities for fulfilling orders.

We created a single page appliation to provide users with a seamless and fast shopping experience. React was used on the front-end in and a Redux store was implemented to facilitate easier access to centralized data to all of the React components.

Local storage was used so that a user's cart would persist even with page refreshes. Usage of local storage also allows for guest users to maintain a cart without logging in.

Backend admin functionality was added to facilitate order fulfillment.

  • Technologies Used:
  • Node.js / Express.js
  • React / Redux
  • PostgreSQL

Chad App

Connecting users through location based chatrooms, both private and public.

Chad is an iOS app currently available in the Apple App Store that was built using React Native.

Chad uses Google Firebase in the backend to store and serve data at lightning speeds. Speed is imperative to the Chad platform to provide users with a quick and responsive messaging experience. Much time was spent crafting a backend architecture that would scale well as the user base grows.

Building the application with React Native allowed myself and the engineering team to create an application that would work on both iOS and Android from one codebase. React Native also made it easy to leverage my experience with React to create reusable components that could be used multiple times throughout the application.

I implemented Redux to provide a central data store for information in the applicaiton. This provided several benefits included reducing the number of API calls made to Firebase as well as reducing application loading times.

  • Technologies Used:
  • React Native
  • Google Cloud Firestore
  • React / Redux

Flashcard Academy

Learn coding on the go, along an optimal timeline. Built entirely in React Native.

Flashcard Academy is a spaced repetition flashcard application where students are quizzed on material along a timeline which optimizes information retention.

Students engage with the app in two main environments: Information Playground and Quizzable Land (cheesy I know...). In the Information Playground users have the ability to choose flashcards by Topic, Difficulty, or Random. Flashcards contain bitsize facts about various software development concepts. They are designed to be short, and digestable, something that you can retain during your commute.

When a student feels confident that they've understood any given card's content they can swipe up to have related questions to that card set as quizzable. If they feel that a card is irrelevant to their studies and they'd choose not to see it again they can swipe down to discard it.

In Quizzable Land students can select quiz cards along Topic, Difficulty, Random, or Timeline to be quizzed on. Quiz cards contain a question and four multiple choice answers. Answering these questions correctly / incorrectly updates a spaced repetition algorithm that determines how comfortable a student is with any concept and adjusts the frequency in which that student is presented with that material.

  • NodeJS / ExpressJS
